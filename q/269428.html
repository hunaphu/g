Haskell + hgl, 13 bytes

cP$h_*>h_>~Êƒ

Attempt This Online!

Explanation

This uses the parser library.

h_ parses any non-empty string, so we ask if there is a h_ followed by h_ and then we feed the output of the second h_ into Êƒ. Since Êƒ parses any fixed string given as input, this means it parses whatever h_ gave again.

cP takes the parser we built and turns it into a function which returns true when there is at least one parse which completely consumes the string.

Reflection

This is such a simple challenge it's hard for this to get any better, but I have some thoughts:

  • (>~Êƒ) is a pretty common function. It attempts to parse the result of the last parse again. However unless I assigned it a 2 byte name it wouldn't make any savings in this particular case.
  • (h_*>) might also be useful to have a shortcut for.
