Encoding format

We store a list of integers representing horizontal words, interleaved with a similar list for vertical words.

Each integer \$q\$ is interpreted as follows:

  • \$n=\lfloor q/2025\rfloor\$ is the index of the word in the dictionary.
  • \$x=\lfloor q/45\rfloor\bmod 45\$ is the column at which the first character is put.
  • \$y=q\bmod 45\$ is the row at which the first character is put.

Each list is encoded as 24-bit delta values and each delta value is encoded as a 4-character string in base-64.

Because most words are taken at the beginning of the dictionary and because the lists are sorted, the 163 first base-64 strings all start with "A". This allows us to compress further by storing the leading characters separately.

Decoding example

  • The first matched string is "T6+".
  • We append the leading "A" from our separate lookup string, which gives "AT6+".
  • We apply base-64 decoding, reduce the 3 resulting bytes to a 24-bit integer (using Big-Endian format) and add \$1\$. This gives \$81599\$.
  • We add this delta value to the current value, which is still \$0\$ at this point. So the result is unchanged: \$q=81599\$.
  • We extract:
    • \$n=\lfloor 81599/2025\rfloor = 40\$ → the corresponding word in the dictionary is "search"
    • \$x=\lfloor 81599/45\rfloor \bmod 45 = 13\$
    • \$y=81599\bmod 45 = 14\$
  • Because we are at an even index, this is a horizontal word. So we write "search" horizontally at \$(13,14)\$ in the matrix.

(Don't) try it online!

Basically, it starts by placing a cross of 2 words of odd length in the center of the grid, then attempts to add connected words, 4 at each step with the required 4-fold rotational symmetry, until 50% of the grid is filled with letters.
