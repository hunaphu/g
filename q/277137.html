Revised answer based on criteria without reflectional symmetry. If the input corresponds to a set of four 1 bits on the faces of the dodecahedron arranged in a Z pattern (or part of such a set) returns false. Otherwise returns true.

The Z pattern has 2 fold rotational symmetry, therefore there are only 30 possible orientations of this pattern on the dodecahedron, and hence 30 possible sets of four 1 bits the input that return falsy. If we were to test for each oriantation separately, then for each orientation of the Z pattern, there would be a 2-fold rotation of the inputs that leaves the output unchanged. Hence there are 60 possible rotations of the inputs that leave the output unchanged.

As a result of golfing, false is also returned for the following additional cases bringing the total to 229: all inputs with 0,1,or 2 1 bits (1+12+66=79); inputs with 3 1 bits corresponding to part of a Z with one of the four bits deleted (4*30=120).

Similar to l4m2's answer, this checks for a particular configuration of 1's on three faces, however a different set of three faces is used: an opposite pair of faces, plus one additional face. Any system of face labelling in which opposite faces have indices 6 apart will do. An example face labelling which I used in a previous challenge is given at the bottom of this answer.

There are 6 ways of choosing a pair of opposite faces, and after that there are 10 ways of choosing the 3rd face, showing that this configuration has 60 possible orientations corresponding to 60 rotations. It has a plane of symmetry (note that configurations without a plane of symmetry have 120 possible orientations if reflections are included.)

The code first checks if a pair of opposite faces have 1's

n&n*64>0

It then converts the number to a binary representation in text and checks if more than 2 bits are set by summing the ascii values and taking the result mod 48

("%b"%n).sum%48>2

The TIO link shows that there 60 possible inputs with exactly three 1 bits set that return true. The code as written will return true if additional faces with 1 bits are present, provided the minimum configuration is met. For cases with more than three 1 bits analysis becomes complicated as there may be several configurations with different symmetry classes that return true. For an additional byte, >2 can be changed to ==3 and the code will then only return true for the 60 inputs that contain exactly the specified configuration of three faces. Another possibility is to change it to <4. This version gives true in 66 cases (the additional 6 being cases where only a pair of opposite faces are selected.)
