Input as a pair of pairs of pairs, where 0 indicates an inclusive range (either [ or ]) and 1 an exclusive range (either ( or )). E.g. (8,9], [1,2) would be input as [[[8,1],[9,0]], [[1,0],[2,1]]].

Port of the approach used in @Neil's Retina answer, so make sure to upvote that answer as well!

Explanation:

Æ      # Reduce each inner-most pair of the (implicit) input by subtracting
       #  e.g. [[[8,1],[9,0]],[[1,0],[2,1]]] → [[7,9],[1,1]]
 ε     # Map over each inner pair:
  Ÿ    #  Convert it to a list in that (inclusive) range
       #   → [[7,8,9],[1]]
   g   #  Pop and push the length
       #   → [3,1]
 }`    # After the map: pop and push both values to the stack
   s@  # Swap them, and do a >= check
       #  → 1>=3 → 0
   ›   # Or for the inverted output: do a > check without swapping
       #  → 3>1 → 1
       # (after which the result is output implicitly)

Uses the legacy version of 05AB1E instead of the new one, since Ÿ on pairs where both ends are the same won't uniquify it (e.g. [1,1] will result in [1] in the legacy version, but [1,1] in the new version, requiring an additional explicit Ùniquify).
