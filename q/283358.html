Encoding

Each element name is broken down into:

  • a 2-byte prefix
  • a 3rd byte in the range \$[128\dots191]\$
  • an optional 4th byte.

There are 16 distinct prefixes.

Hence the following encoding on 10 bits:

vvvvvvpppp
\____/\__/
   |    |
   |    '--> prefix identifier
   '-------> value of 3rd byte, minus 128

This 10-bit value is encoded as 2 characters in base 36. Special values higher than 1024 are used for repeated whitespace.

If the prefix identifier is either 14 or 15, a 4th byte is taken from a separate lookup table.
