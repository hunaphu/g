Uses space for white, 0 for grey and 1 for black. It's a little difficult to see the boundary between the first and second grids, but the boundary above 3rd and 4th grid is clear.

We print a grid of characters 18 wide x 17 high for each stage, total 68 lines. The fact that the grids are 17 high means the first and last line of each grid are the same, and every fourth line of the whole output is all zeros, with the data shifted one cell right every 17 lines.

The black/white cells to plot comprise 17 rows and 5 columns of data, the leftmost of which is always 0. Each of the 17 characters in the magic string represents 1 row. Bits are extracted and multiplied by 14, giving the binary representation " 0" for 0 and "1110" for 1. The 0 forms the right wall of the cell.

For each line we compile a string of 28 bits (including the 1 from the 64's bit which is always set for characters in the selected range of the magic string) and pick 18 to print by [j/17,18]

Using 68.times{|j|puts...} is shorter overall than returning from a function with ->{(0..67).map{|j|...}}. Changing puts to p would save 3 bytes but add " at the beginning and end of each line.
