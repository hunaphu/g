I'll go through this line-by-line.

{
  {

Starts two closures. The inner one is used to be able to refer to a value (passed as an argument) multiple times without the overhead of a let statement. (This also means we can put the entire thing in a single expression, allowing for an implicit return.)

    Int($0.1)! < 24 && Int($0.2)! < 60

First, we check that the hour part of the clock ($0.1) is less than 24 and that the minute part ($0.2) is less than 60.

$0 here is of type (Substring, Substring, Substring, Substring). $0.0 is the entire input (unused here); $0.1 is the hour; $0.2 is the minute; and $0.3 is everything after the minute.

      ? (1...12 ~= Int($0.1)! ? 1 : 0)
        + ($0.3 > "" ? 0 : 1)

If the hour and minute are in range, then we do two things:

  • Check if the hour is within the range 1...12 (inclusive on both ends) by abusing the pattern matching operator.
  • Check if there's anything after the minute (i.e. AM/PM has been specified) by comparing to an empty string.

Each of these checks returns 1 if it succeeds, or 0 if not. We then add these together and return it as the final score.

This bit was adapted from @Arnauld's JavaScript answer; I've reproduced his scoring table here for reference.

1...12 ~= hour suffix == "" Score Examples
false (0) false (0) 0 14:00 PM, 00:00 PM
false (0) true (1) 1 14:00, 00:00
true (1) false (0) 1 07:00 PM
true (1) true (1) 2 07:00 
      : 0

If the hour and/or minute are not in range, return 0 -- this indicates an invalid clock.

  }(

End the inner closure and immediately call it.

    ($0 + "").matches(of: /(.+):(..)(.*)/)[0]

We grab the outer closure parameter (i.e. the actual input string) and extract data using a regex. (The match will always succeed.)

  • The first capture group picks up one or more characters; in our case, it's always either 1 digit or 2. This is the hour.
  • After matching a colon, the next capture group matches exactly 2 characters. This is the minute.
  • The final capture group sucks up whatever's left, if anything. If AM/PM was specified in the input, this will match " AM" or " PM"; if it wasn't specified, this simply matches the empty string.

The match result is passed directly to the inner closure, which can use dynamic member lookup to access the captures.

$0 + "" serves as a hint to the type checker that $0 is of type String.

  ) as Int
}

Finally, implicitly return the result of the inner closure. as Int is necessary as a type checker hint (though Swift has successfully type-checked far more complicated programs of mine without any as hints, so why it's needed here is beyond me.)

How?

A clock will be right \$\binom{N_{24}+N_{12}}{K}\$ times, where:

  • \$N_{24}\$ is \$1\$ if the hour and minute can occur on a twenty-four-hour clock, else \$0\$
  • \$N_{12}\$ is \$1\$ if the hour and minute can occur on a twelve-hour clock, else \$0\$
  • \$k\$ is \$2\$ if an AM or PM is present, else \$1\$

Note that \$N_{12}\$ can only ever be \$1\$ if \$N_{24}\$ is \$1\$, so all cases are:

N24 + N12 K (N24 + N12)-choose-K
1 + 1 2 1
1 + 1 1 2
1 + 0 2 0
1 + 0 1 1
0 + 0 2 0
0 + 0 1 0
