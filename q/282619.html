Function returning a string. X & + are rendered as 0 & 2 as discussed in the comments.

The code starts with an array of strings in %w{} format. Three characters at the beginning each line encode bitwise (in the six rightmost bits) for "space" or "not space" in the 18 columns on the right of the output. Note that the first string contains . instead of space since whitespace is used as a separator in %w{} format.

The "not spaces" are then filled in with a hardcoded string for column labels plus a base 3 number for the table data (compressed to base 36). Two character transpositions are required in the final output: . is changed to space and 1 (from the base 3 number) is changed to -.

Encoding

Each row is encoded as:

  • a base-36 value storing the types for this row (2 bits per type)
  • immediately followed by the label

All rows are joined together into a single data string:

      +---------------------> base-36 value
      |     +---------------> label
      |     |     +---------> base-36 value
      |     |     |      +--> label
    __|_  __|_  __|__  __|_
   /    \/    \/     \/    \
...28ovf2Poison4836scoGround...

The fields are extracted with the following regular expression:

/([a-z\d]*)(\D+)/g

The transitions value -> label are triggered by a letter in upper case.

The transitions label -> value are triggered by a digit. Most values start with a digit but we occasionally have to add a leading zero, as in:
